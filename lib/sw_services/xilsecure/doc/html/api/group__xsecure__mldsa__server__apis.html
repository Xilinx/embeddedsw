int XSecure_MldsaSignGenerate ( XSecure_MldsaSignGenParams *  MldsaParams)

This function generates MLDSA signature for the given data using the provided secret key.

Parameters
MldsaParamsPointer to MLDSA signature generation parameters instance.
Returns
  • XST_SUCCESS On successful signature generation.
  • Error code On failure.
  • Validate MLDSA instance pointer
  • Validate secret key length
  • Validate signature buffer length
  • Generate signing randomness using TRNG (32 bytes)
  • Generate entropy using TRNG (64 bytes)
  • Release MLDSA from reset
  • Wait for the core to be ready (STATUS flag should be 2'b01 or 2'b11)
  • Feed the required inputs: msg, sk_in, sign_rnd, entropy
  • Transfer secret key to hardware registers
  • Transfer signing randomness to hardware registers (256 bits = 32 bytes)
  • Transfer entropy to hardware registers (512 bits = 64 bytes)
  • Trigger the core for performing Signing (STATUS flag will be changed to 2'b00)
  • Configure ML-DSA for signature generation with streaming message mode
  • Wait for the core to be ready to receive the message in streaming mode
  • Transfer message/data to hardware registers
  • Wait for the core to be ready and valid (STATUS flag should be 2'b11)
  • Read the generated signature from hardware registers
  • Transfer word-aligned portion (4624 bytes = 1156 words)
  • Handle last 3 bytes using intermediate word buffer
  • Clear ML-DSA control registers (using ZEROIZE command) to remove sensitive data from hardware for security
  • Wait for zeroize operation to complete
  • Put MLDSA into reset state

int XSecure_MldsaSignVerify ( const XSecure_MldsaSignVerifyParams *  MldsaParams)

This function verifies MLDSA signature for the given data using the provided public key and context.

Parameters
MldsaParamsPointer to MLDSA signature verification parameters instance.
Returns
  • XST_SUCCESS On successful verification.
  • Error code On failure.
  • Validate MLDSA instance pointer
  • Validate ctx length
  • Check that first 64 bytes of signature are not all zeros (CT validation)
  • Release MLDSA from reset
  • Wait for the core to be ready for processing inputs
  • Write context length to context config register
  • Transfer context to hardware registers
  • Transfer public key to hardware registers
  • Transfer signature to hardware registers
  • Transfer word-aligned portion (4624 bytes = 1156 words)
  • Handle last 3 bytes using intermediate word buffer
  • Configure ML-DSA for signature verification with streaming message mode
  • Wait for the core to be ready to receive the message in streaming mode
  • Transfer message/data to hardware registers
  • Wait for signature verification operation to complete
  • Compare generated result with expected signature's CT component
  • Clear ML-DSA control registers (using ZEROIZE command) to remove sensitive data from hardware for security
  • Wait for zeroize operation to complete
  • Put MLDSA into reset state
