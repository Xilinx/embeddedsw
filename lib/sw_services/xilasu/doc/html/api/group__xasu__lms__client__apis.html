s32 XAsu_HssSignVerify ( XAsu_ClientParams *  ClientParamsPtr,
XAsu_LmsHssSignVerifyParams *  HssParamsPtr 
)

This function sends a command to ASUFW to perform HSS (Hierarchical Signature System) signature verification on the provided message using the given HSS public key.

Parameters
ClientParamsPtrPointer to the XAsu_ClientParams structure which holds the client input parameters.
HssParamsPtrPointer to XAsu_LmsHssSignVerifyParams structure which holds the parameters for HSS signature verification.
Returns
  • XST_SUCCESS, if IPI request to ASU is sent successfully.
  • XASU_INVALID_ARGUMENT, if any argument is invalid.
  • XASU_QUEUE_FULL, if Queue buffer is full.
  • XST_FAILURE, if sending an IPI request to ASU fails.
  • XASU_INVALID_UNIQUE_ID, if received Queue ID is invalid.

Validate input parameters.

Generate a unique ID and register the callback function.

Set the command ID based on the selected SHA type.

Create command header.

Update request buffer and send an IPI request to ASU.

s32 XAsu_LmsSignVerifyKat ( void  )

This function runs LMS signature verification KAT from the client side. 

    It sends an LMS sign verify request to ASUFW using known good test vectors,
    busy-waits for the response, and returns the verification result.
Returns
  • XST_SUCCESS, if LMS signature verification KAT passes.
  • XST_FAILURE, if the KAT fails.

Configure LMS signature verification parameters using KAT vectors.

Configure client parameters for synchronous operation.

Send LMS signature verification request to ASUFW.

Busy-wait for ASUFW response.

Check verification status from callback.
