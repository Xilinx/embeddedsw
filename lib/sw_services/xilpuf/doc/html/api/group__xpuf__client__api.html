int XPuf_ClearPufID ( const XPuf_ClientInstance *  InstancePtr)

This function sends IPI request for clearing PUF ID.

Parameters
InstancePtrPointer to the client instance
Returns
  • XST_SUCCESS if the PUF clear ID is successful
  • XST_FAILURE if there is a failure
  • Perform input parameter validation on InstancePtr. Return XST_FAILURE if input parameters are invalid.
  • Fill IPI Payload
  • Send request to server via SMC for EL1 non-secure AArch64 applications or via IPI mailbox for other environments.

int XPuf_ClientInit ( XPuf_ClientInstance *const  InstancePtr,
XMailbox *const  MailboxPtr 
)

This function sets the instance of mailbox.

Parameters
InstancePtrPointer to the client instance
MailboxPtrPointer to the mailbox instance
Returns
  • XST_SUCCESS on successful initialization
  • XST_FAILURE on failure
  • Set XMailbox instance provided by the user to client library instance by validating whether provided instance is initialized and not NULL. This XMailbox instance will be used by client library to send requests to PLM.

int XPuf_SendRequest ( const XPuf_ClientInstance *  InstancePtr,
u32 *  PayloadBuf,
u32  PayloadLen,
u32 *  ResponseBuf,
u32  ResponseLen 
)

This function sends request to PLM through either SMC or IPI mailbox based on the execution environment.

For EL1 non-secure AArch64 applications, SMC is used. For other environments, IPI mailbox is used.

Parameters
InstancePtrPointer to the client instance
PayloadBufPointer to the payload message
PayloadLenLength of the payload message
ResponseBufPointer to store response values (can be NULL)
ResponseLenLength of response buffer (number of u32 elements)
Returns
  • XST_SUCCESS if the request is successful
  • XST_FAILURE if there is a failure
  • XST_INVALID_PARAM if input parameters are invalid
Note
This is a generic API that internally handles SMC or Mailbox communication based on the build configuration.
  • Validate input parameters.
  • For other environments, use IPI mailbox communication.
  • Validate mailbox pointer for IPI communication
  • Send request through IPI mailbox
  • Note: For communication via IPI mailbox, response is not directly returned. The caller should handle response retrieval if needed.

References XPuf_ClientInstance::MailboxPtr, and XPuf_ProcessMailbox().
