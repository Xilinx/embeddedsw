u32 *  MsgPtr,

)

This function sends IPI request to the target module and gets the response from it.

Parameters
MailboxPtrPointer to mailbox instance
MsgPtrPointer to the payload message
MsgLenLength of the message
Returns
  • XST_SUCCESS If the IPI send and receive is successful
  • XST_FAILURE If there is a failure
Note
Payload consists of API id and call arguments to be written in IPI buffer
  • Send IPI CDO to PLM. Return XST_FAILURE, if failure in IPI send request.
  • Wait for IPI response from PLM with a default timeout of 2 seconds. If the timeout exceeds, then error is returned otherwise it returns the status of the IPI response.

int XNvm_SendRequest ( const XNvm_ClientInstance *  InstancePtr,
u32 *  PayloadBuf,
u32  PayloadLen,
u32 *  ResponseBuf,
u32  ResponseLen 
)

This function sends request to PLM through either SMC or IPI mailbox based on the execution environment.

For EL1 non-secure AArch64 applications, SMC is used. For other environments, IPI mailbox is used.

Parameters
InstancePtrPointer to the client instance
PayloadBufPointer to the payload message
PayloadLenLength of the payload message
ResponseBufPointer to store response values (can be NULL)
ResponseLenLength of response buffer (pass 0U when ResponseBuf is NULL)
Returns
  • XST_SUCCESS If the request is successful
  • XST_FAILURE If there is a failure
  • XST_INVALID_PARAM If input parameters are invalid
Note
This is a generic API that internally handles SMC or Mailbox communication based on the build configuration.
  • Validate input parameters
  • Validate mailbox pointer for IPI communication
  • Send request through IPI mailbox

Note: For mailbox path, response is not directly returned. The caller should handle response retrieval if needed.
