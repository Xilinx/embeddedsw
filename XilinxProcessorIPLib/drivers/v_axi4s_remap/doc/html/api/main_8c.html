int driverInit ( )

Initialize video pipeline driver instances.

This function initializes the VTC, TPG, and AXI4S Remap driver instances by looking up their configurations and performing configuration initialization. All three drivers must be successfully initialized for the video pipeline to function correctly.

Returns
XST_SUCCESS if all drivers initialized successfully XST_DEVICE_NOT_FOUND if any device configuration lookup fails XST_FAILURE if any driver initialization fails
Note
This function must be called before using any video IP functions

int main ( )

Main test application for video pipeline.

This is the main entry point for the video pipeline test application. It initializes all video driver instances, tests three video modes (1080p60, 4K30, and optionally 4K60), and verifies video lock for each mode. The test validates proper operation of the complete video processing chain.

Returns
XST_SUCCESS (0) if all tests pass XST_FAILURE if any initialization or test fails
Note
4K60 test is only executed if the IP is configured for 2 or 4 pixels per clock to support the required bandwidth

References XV_axi4s_remap::Config, driverInit(), XV_axi4s_remap_Config::PixPerClkOut, resetIp(), videoClockConfig(), and videoIpConfig().

void resetIp ( void  )

Reset HLS IP cores in the video pipeline.

This function performs a hardware reset of the HLS IP cores by asserting and then releasing the reset signal through GPIO. It includes appropriate delays to ensure the reset is properly applied and released.

Returns
None
Note
Reset signal is active low (0 = reset asserted, 1 = reset released)

int videoClockConfig ( XVidC_VideoMode  videoMode)

Configure video clock generator for specified video mode.

This function configures the programmable video clock generator to produce the appropriate pixel clock frequency for the specified video mode. It calculates and programs the PLL divider and multiplier values, then waits for the PLL to achieve lock.

Parameters
videoModeVideo mode enumeration specifying the target resolution and frame rate for clock generation
Returns
XST_SUCCESS if clock generator configured and locked successfully XST_FAILURE if unsupported video mode or PLL lock timeout
Note
Supports 1080p60, 4K30, and 4K60 video modes with 1, 2, or 4 pixels per clock configurations

void videoIpConfig ( XVidC_VideoMode  videoMode)

Configure video IP cores for specified video mode.

This function configures the AXI4S Remap, TPG, and VTC IP cores with timing parameters appropriate for the specified video mode. It sets resolution, color format, pixel clock parameters, and timing generator settings.

Parameters
videoModeVideo mode enumeration specifying the desired resolution and frame rate (e.g., 1080p60, 4K30, 4K60)
Returns
None
Note
The video clock generator must be configured and locked before calling this function
