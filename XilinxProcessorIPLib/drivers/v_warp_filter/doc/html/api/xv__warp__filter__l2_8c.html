void XVWarpFilter_ClearNumOfDescriptors ( XV_warp_filter *  InstancePtr)

Clears and de-allocates all descriptors.

This function traverses the descriptor chain and frees all allocated memory for each descriptor, then resets the base address and count.

Parameters
InstancePtris a pointer to the XV_warp_filter instance.
Returns
None.
Note
This function must be called before the instance is destroyed to prevent memory leaks.

Programs a descriptor with given configurations.

This function locates the specified descriptor and configures it with the provided input parameters including dimensions, addresses, and processing parameters.

Parameters

configPtris the input configuration pointer containing the configuration parameters.
valid_segis the number of valid segments in the remap vector.
lblock_countis the number of output blocks decoded by the first warp filter core.
line_numis the line number from which the second warp filter core starts reading the source image.
Returns

Note

s32 XVWarpFilter_SetNumOfDescriptors ( XV_warp_filter *  InstancePtr,
u32  num_descriptors 
)

Creates the descriptors for the warp filter.

This function allocates and initializes a chain of descriptors for the warp filter operation. Each descriptor is aligned to the address width requirement and linked together in a chain.

Parameters
InstancePtris a pointer to the XV_warp_filter instance.
num_descriptorsis the number of descriptors to be created.
Returns
XST_SUCCESS if descriptors created successfully. XST_FAILURE if descriptor creation failed.
Note
Descriptors are allocated with aligned memory and linked in a chain.

s32 XVWarpFilter_Stop ( XV_warp_filter *  InstancePtr)

Stops the core instance.

This function stops the warp filter core by clearing the autostart bit, setting the flush bit, and waiting for the flush operation to complete.

Parameters
InstancePtris a pointer to the XV_warp_filter instance.
Returns
XST_SUCCESS if the core entered stop state successfully. XST_FAILURE if the core failed to stop (flush timeout).
Note
The function waits for a flush done signal with timeout.
