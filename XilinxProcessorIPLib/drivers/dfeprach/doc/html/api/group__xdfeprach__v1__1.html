u32 XDfePrach_AddCC ( XDfePrach *  InstancePtr,
s32  CCID,
u32  SlotSeqBitmap,
const XDfePrach_CarrierCfg *  CarrierCfg 
)

Adds specified CCID, with specified configuration.

Parameters
InstancePtris a pointer to the PRACH instance.
CCIDis a Channel ID.
SlotSeqBitmap- up to 16 defined slots into which a CC can be allocated. The number of slots can be from 1 to 16 depending on system initialization. The number of slots is defined by the "sequence length" parameter which is provided during initialization. The Bit offset within the SlotSeqBitmap indicates the equivalent Slot number to allocate. e.g. 0x0003 means the caller wants the passed component carrier (CC) to be allocated to slots 0 and 1.
CarrierCfgis a CC configuration container.
Returns
  • XST_SUCCESS if successful.
  • XST_FAILURE if error occurs.

References XDfePrach_CCCfg::CarrierCfg, XDfePrach_CarrierCfg::Enable, XDfePrach_CCCfg::Sequence, XDfePrach::StateId, and XDFEPRACH_STATE_OPERATIONAL.
